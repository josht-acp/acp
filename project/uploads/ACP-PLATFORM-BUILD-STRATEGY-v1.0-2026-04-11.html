Strategic Framework — Waterfall + Agile Hybrid

Waterfall: phase gates (macro)
  • Each phase has a hard entry criterion and a hard exit deliverable
  • No phase begins until prior gate is formally cleared
  • Gates prevent expensive rework — architecture locks before build
  • Sequence: Discovery → Architecture → Foundation → Build → QA/UAT → Launch → Scale
  • Matches ACP PMO model: Workstream → Epic → Story → Task
  • Irreversible decisions (DB schema, auth model, domain) made at Architecture gate only
Agile: sprint delivery (micro)
  • 2-week sprints inside each waterfall phase
  • Daily async standup via Notion task update (solo execution)
  • Sprint backlog pulled from prioritised epic story map
  • Story points: Fibonacci (1, 2, 3, 5, 8) — max 21 pts/sprint
  • Velocity tracked sprint-over-sprint to forecast phase completion
  • Retrospective every sprint: one thing to stop, one to start
Build sequence logic
  • Internal first, external second — faster feedback, lower risk
  • Shared infrastructure built once (auth, DB, API, design system)
  • Executive Reporting unlocked first — highest internal ROI
  • WorkHub/CRM second — team enablement before client exposure
  • LP Portal third — revenue-facing, requires auth from Phase 1
  • Public Website last — marketing, not revenue-blocking
Consolidation principle
  • Everything existing gets integrated, not rebuilt
  • Stripe Portal → embedded in LP Portal fee section
  • Google Forms Blueprint → migrated to WorkHub intake engine
  • Notion Execution OS → synced via API, not replaced
  • ACP Template OS (140+ templates) → powering Document Generator
  • Deck Library → embedded in LP Portal document rooms

Technology Stack — Rationale-driven

Framework
Next.js 14 (App Router)
One codebase for all 4 platforms. SSR for LP Portal. RSC for dashboard performance.
Auth
Supabase Auth + MFA
Row-level security. Built-in SSO. Per-mandate scoped permissions. Audit logs.
Database
PostgreSQL + Prisma
Relational = correct for deal/fee/investor data. Prisma type-safety. Supabase hosted.
Payments
Stripe (already built)
9 SKUs live in sandbox. Portal built. Just embed + go live. Zero rebuild.
Notion sync
Notion API + webhooks
Notion stays SOR. Platform pulls/writes via API. No migration required.
Documents
DocuSign + Gamma + Canva
DocuSign for binding signatures. Gamma for auto-deck generation. Canva for polish.
Hosting
Vercel + Cloudflare
Edge functions for LP Portal. CDN + WAF for website. Auto SSL. Global low-latency.
Comms
Resend + Google Workspace
Resend for transactional email. Gmail API for CRM sync.
Analytics
PostHog + GA4
PostHog for product analytics (LP Portal). GA4 for website traffic.
Charts
Recharts + D3
Recharts for standard dashboard charts. D3 for pipeline funnel + fee waterfall.
CMS
Sanity.io
Public website content managed without deploys. Structured content + image pipeline.
Security
OWASP + Snyk + Pentest
Required before LP Portal launch. Snyk for dependency scanning. External pentest W13.
